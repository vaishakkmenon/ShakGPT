// Architecture
2
Model Config + ModelConfig dataclass
Single source of truth for all hyperparameters
Up next
Before writing any neural network code, define the ModelConfig dataclass. Every component reads from this — changing one value updates the entire model. Key values: d_model=1024, n_layers=24, n_heads=16, n_kv_heads=8, ffn_hidden=2752, max_seq_len=16384, vocab_size=32768, rope_theta=500000.
○
Write ModelConfig in model/config.py
Use Python dataclass decorator. Define all architecture hyperparameters with correct defaults. Understand what each value means before writing it.
Build
Learning note: A config is a single source of truth. Every component takes a config object instead of hardcoded numbers. Changing d_model in one place updates the entire model automatically.
3
RMSNorm
Normalisation layer — stability mechanism between blocks
1–2 hrs
Simplest component. Rescales token vectors to keep values in a stable range as they flow through 24 layers. Without it values explode or vanish by layer 20. Pre-norm placement means it runs before attention and before FFN in each block.
○
Implement RMSNorm as nn.Module
Divides by root mean square of the vector. One learnable scale parameter (gamma). No bias (beta) needed. eps=1e-5 for numerical stability.
Build
4
RoPE — Rotary Positional Encoding
Gives the model a sense of token position and distance
2–4 hrs
Without positional encoding, "dog bites man" and "man bites dog" look identical to attention — same tokens, different order. RoPE rotates token vectors by an angle proportional to their position. Tokens close together get similar rotations, tokens far apart get very different rotations. theta=500000 (LLaMA-3 value) enables good generalisation at 16K context.
○
Precompute cos/sin frequency tables
Compute once at model init and register as buffers so they move with .to(device). Shape: (max_seq_len, head_dim // 2).
Build
○
apply_rope function
Splits head_dim in half, applies rotation to query and key vectors. Applied inside attention, not to value vectors.
Build
5
Grouped Query Attention (GQA)
The core mechanism — tokens talk to each other
1–2 days
The most complex component. Each token produces Query, Key, Value vectors via learned weight matrices. Query/Key dot products produce attention scores — how much each token should attend to every other token. GQA uses 16 query heads but only 8 KV heads, halving KV cache size at 16K context. Flash Attention 2 via F.scaled_dot_product_attention handles the efficient computation.
○
Q, K, V projections (no bias)
q_proj: d_model → n_heads * head_dim. k_proj and v_proj: d_model → n_kv_heads * head_dim. This asymmetry is the GQA part.
Core
○
Apply RoPE to Q and K
Not to V — positional encoding only needed for the matching step, not the value aggregation.
Core
○
Expand KV heads to match Q heads
repeat_interleave by n_groups (=2) along head dim before SDPA. This is how GQA makes 8 KV heads serve 16 query heads.
Core
○
F.scaled_dot_product_attention
PyTorch's Flash Attention 2 implementation. is_causal=True for training. Handles the Q*K/sqrt(d)*V computation efficiently.
Core
○
KV cache (modular)
Keep it in a clean swappable module. Used at inference only. This is where PolarQuant compression slots in later.
Inference
6
SwiGLU Feed-Forward Network
Each token processes what it learned from attention
2–4 hrs
After attention tokens have gathered information from neighbours. The FFN is where each token independently processes that information. Expands to 2752 dimensions (8/3 * d_model, rounded to multiple of 64 for GPU efficiency), applies SwiGLU activation, compresses back. This is where most of the model's factual knowledge is stored.
○
Three weight matrices — gate, up, down
SwiGLU uses 3 projections instead of standard 2. gate and up both project d_model → 2752. down projects 2752 → d_model. No bias on any of them.
Build
○
SwiGLU forward: down(SiLU(gate(x)) * up(x))
The gating mechanism — SiLU(gate) acts as a learned filter on up(x). This is what makes SwiGLU outperform standard GELU FFN.
Build
7
TransformerBlock + Full LLM
Wire everything together into a complete model
1 day
TransformerBlock wires RMSNorm → GQA → residual → RMSNorm → FFN → residual. The full LLM stacks 24 of these blocks, adds token embeddings, a final RMSNorm, and the output projection. Weight tying shares embedding and output projection weights, saving 33M parameters.
○
TransformerBlock — pre-norm pattern
Norm before each sub-layer, residual connection after. x = x + attn(norm(x)), then x = x + ffn(norm(x)).
Build
○
LLM — embeddings + 24 layers + lm_head
Weight tie embed and lm_head. Register RoPE buffers. Scaled init on o_proj and down_proj (1/sqrt(2*n_layers)) to prevent residual stream explosion.
Build
○
Sanity check — random input loss ≈ 10.4
log(32768) = 10.4. A randomly initialised model on random data should produce this loss. If not, something is broken in the forward pass.
Verify

Data Pipeline

1–2 weeks
Unified preprocessing pipeline that normalises encoding, whitespace, and formatting across all four sources before tokenization. Stream directly to binary shards to avoid local storage issues. Test on 1M token sample before processing full corpus.

UTF-8 encoding, NFC unicode normalisation, CRLF → LF, max 2 consecutive newlines, minimum 100 word filter. All sources go through the same funnel.

Stream + tokenize + pack to binary shards
100M tokens per shard as uint16 .bin files. Never save intermediate jsonl if storage is limited. Enforce 60/20/15/5 ratio at chunk level.

// Instrumentation (built during phases 7–10)
R1
Hidden state extraction hooks
Extract any layer's output during any forward pass
Build with Phase 7
Add PyTorch forward hooks to every TransformerBlock that optionally save the hidden state output. When enabled, a single forward pass returns not just logits but a list of 24 tensors — one per layer. This is the foundation for everything else in the research track.
○
register_forward_hook on each layer
Controlled by a flag so it adds zero overhead during training. Activates only during analysis runs.
Instrument
R2
Attention weight logging
Save attention matrices for visualisation
Build with Phase 5
Optionally return attention weight matrices from GroupedQueryAttention during inference. These are the (n_heads, T, T) matrices that show which tokens attended to which. The raw material for attention heatmap visualisations.
○
return_attn_weights flag in attention forward
When True, return the softmax weights before the value aggregation step. Shape: (B, n_heads, T, T).
Instrument
R3
Per-source loss tracking
Watch when the model learns each type of content
Build with Phase 10
Instead of logging a single loss number, log loss separately for held-out samples from each data source. Plotting these over training shows exactly when the model started learning web text vs books vs code vs Wikipedia — and whether learning one thing helped or hurt another.
○
Maintain separate val sets per source
Hold out 1000 examples from each source before training. Evaluate every 500 steps. Log to wandb as separate loss curves.

// Analysis (after training)

Attention visualisation dashboard
See which tokens attend to which across all layers
Post-training
Interactive heatmap visualisation. Input any sentence, select a layer and head, see the attention weight matrix as a colour grid. Watch how attention patterns change from early layers (often positional/syntactic) to later layers (semantic relationships). BertViz or TransformerLens as the backbone.
○
Integrate TransformerLens or BertViz
TransformerLens is purpose-built for this — hook into any layer, patch activations, run causal analysis. Adapt to your architecture.

Hidden state PCA visualisation
Watch representations form across layers
Post-training
Extract hidden states at each layer for a set of sentences from different domains. Use PCA to compress 1024 dimensions to 2D and plot. Colour by domain — medical text, general text, code. Watch whether the model separates them and at which layer that separation emerges.

R6
Probing classifiers
Find where specific knowledge lives in the network
Post-training
Train tiny linear classifiers on top of frozen hidden states at each layer. Ask questions like "does this layer know if this is a medical sentence?" or "does this layer encode part of speech?". If the classifier works, that layer has encoded that information. Plotting accuracy across 24 layers shows exactly where knowledge lives.
○
Linear probe per layer per concept
Single nn.Linear trained on frozen hidden states. Concepts to probe: domain classification, part of speech, sentence boundary, medical vs general text.
